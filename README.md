# Hootly

Paste a GitHub repo URL, get an interactive onboarding guide. Hootly analyses the codebase with Claude and produces an architecture overview, key file rankings, dependency graph, and a suggested reading order — so you can get up to speed on any repo in minutes.

![Hootly screenshot](frontend/public/logo.svg)

## Features

- **Architecture overview** — tech stack, entry points, key directories, design patterns
- **Key files** — ranked by importance with explanations and key exports
- **Dependency graph** — interactive D3 force graph, colour-coded by language or directory
- **Reading order** — step-by-step guide through the codebase
- **Onboarding guide** — plain-English walkthrough generated by Claude
- **Caching** — re-visiting the same commit skips the pipeline entirely
- **Export** — download as Markdown or print to PDF
- **Auth** — email/password + GitHub OAuth
- **Plans** — Free (1 analysis/month) and Pro (unlimited, larger repos)
- **Usage analytics** — per-user activity charts and top-repo stats

## Tech stack

| Layer | Tech |
|---|---|
| Frontend | Next.js 15 (Pages Router), TypeScript, Tailwind CSS |
| Backend | Python 3.11+, FastAPI, SQLAlchemy, SQLite |
| AI | Anthropic Claude API (4-pass analysis pipeline) |
| Auth | PyJWT, bcrypt, GitHub OAuth |
| Payments | Stripe Checkout + webhooks |
| Email | Resend API |

## Prerequisites

- Python 3.11+
- Node.js 18+
- An [Anthropic API key](https://console.anthropic.com/)

Everything else (Stripe, GitHub OAuth, email) is optional for local development.

## Quick start

```bash
# 1. Clone
git clone https://github.com/Hootly-Labs/hootly.git
cd hootly

# 2. Configure
cp backend/.env.example backend/.env
# Open backend/.env and set ANTHROPIC_API_KEY at minimum

# 3. Run
./start.sh
```

The script installs dependencies and starts both servers:

- Frontend: http://localhost:3000
- Backend API: http://localhost:8000

## Manual setup

### Backend

```bash
cd backend
python -m venv .venv && source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env          # then fill in ANTHROPIC_API_KEY
uvicorn main:app --reload
```

### Frontend

```bash
cd frontend
npm install
# optional: cp .env.local.example .env.local
npm run dev
```

## Environment variables

Copy `backend/.env.example` to `backend/.env`. Only `ANTHROPIC_API_KEY` is required to run locally.

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | Your Anthropic API key |
| `JWT_SECRET` | Yes | Long random string for signing JWTs |
| `DATABASE_URL` | No | Defaults to `sqlite:///./hootly.db` |
| `ALLOWED_ORIGINS` | No | CORS origins, default `http://localhost:3000` |
| `GITHUB_CLIENT_ID` | No | GitHub OAuth app client ID |
| `GITHUB_CLIENT_SECRET` | No | GitHub OAuth app client secret |
| `STRIPE_SECRET_KEY` | No | Stripe secret key (payments) |
| `STRIPE_WEBHOOK_SECRET` | No | Stripe webhook signing secret |
| `STRIPE_PRO_PRICE_ID` | No | Stripe price ID for the Pro plan |
| `RESEND_API_KEY` | No | Resend API key for email notifications |
| `APP_URL` | No | Public URL of the app, used in emails |

## GitHub OAuth setup (optional)

1. Go to GitHub → Settings → Developer settings → OAuth Apps → New OAuth App
2. Set **Homepage URL** to `http://localhost:3000`
3. Set **Authorization callback URL** to `http://localhost:8000/api/auth/github/callback`
4. Copy the client ID and secret into `backend/.env`

## Stripe setup (optional)

1. Create a product and recurring price in the Stripe dashboard
2. Copy the price ID (`price_...`) into `STRIPE_PRO_PRICE_ID`
3. Run the Stripe CLI to forward webhooks locally:
   ```bash
   stripe listen --forward-to localhost:8000/api/billing/webhook
   ```
